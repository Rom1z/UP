# Регламент еженедельного обслуживания сервера и рабочих станций

## Цель
Обеспечение стабильной работы ИТ-инфраструктуры, своевременное выявление и устранение неполадок.

## Периодичность
Каждую пятницу после окончания рабочего дня (или в другое согласованное время).

## Перечень работ

### 1. Проверка журналов событий
- На сервере и критически важных рабочих станциях открыть **Просмотр событий**.
- Просмотреть журналы **Система**, **Приложение** на наличие ошибок и предупреждений.
- Зафиксировать необычные события в отчёте.

### 2. Контроль свободного места на дисках
- Проверить свободное место на всех дисках (особенно системных).
- При остатке менее 15% принять меры (очистка диска, перенос данных, расширение тома).

### 3. Обновление антивирусных баз
- Убедиться, что антивирусное ПО (Kaspersky Endpoint Security) обновило базы.
- При необходимости запустить обновление вручную.

### 4. Тестовое резервное копирование
- Создать тестовую теневую копию папки `\\SRV-FILE\Docs`.
- Проверить возможность восстановления одного файла.

### 5. Проверка доступности общих ресурсов
- С любой рабочей станции попытаться подключиться к общим папкам сервера.
- Проверить права доступа для разных пользователей.

### 6. Анализ производительности
- Оценить загрузку CPU и памяти на сервере (диспетчер задач).
- При аномалиях использовать Performance Monitor для детального анализа.

## Отчётность
Результаты обслуживания заносятся в **Журнал администратора** (файл Excel или текстовый документ) с указанием даты, выполненных работ и выявленных проблем.

## Ответственный
Системный администратор.
